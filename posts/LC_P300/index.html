🟡 300. Longest Increasing Subsequence

LIS 是研究所考試常見的題目，但參考書提供的考試寫法在實際使用上有一些限制，在這題直接使用會 WA 。在這裡以LeetCode 300. Longest Increasing Subsequence 為例，紀錄了時間複雜度從 O(n2)O(n^2)O(n2) 到 O(nlogn)O(nlogn)O(nlogn) 的三種解法，並在題目要求外，額外添加了還原 LIS 的過程。

由於題目要求的是 LIS 的長度，因此還原部分的程式碼未經完全測試，僅供參考。若有錯誤之處，還請斧正。

方法一：視為LCS問題後，Dynamic Programming (in Textbook)

這裡使用了在 CLRS Exercise 14.4-5 的 Solution 中給出的方法：

  • 令 XXX 為原序列，YYY 為原序列排序後所得的序列，
  • XXX 和 YYY 的 Longest Common Subsequence (LCS)，就是 XXX 的 Longest Increasing Subsequence (LIS)。
    • 這裡把 XXX 擺在 左邊，YYY 擺在 上面，方便理解。
  • 但直接使用這種方法，若遇到數字重複的的情況，就會出錯，因此要先對 YYY 去重 。
  • 紀錄LCS時，使用了兩個二維陣列， dpdpdp 表示 LCS 的長度， labellabellabel 表示 LCS 的轉移來源。
    • 若 X[i−1]==Y[j−1]X[i-1] == Y[j-1]X[i−1]==Y[j−1]，則 dp[i][j]=dp[i−1][j−1]+1dp[i][j] = dp[i-1][j-1] + 1dp[i][j]=dp[i−1][j−1]+1，且 label[i][j]="↖"label[i][j] = "↖"label[i][j]="↖"。代表 X[i−1]X[i-1]X[i−1] 和 Y[j−1]Y[j-1]Y[j−1] 相等，可以接在 X[:i−1]X[:i-1]X[:i−1] 和 Y[:j−1]Y[:j-1]Y[:j−1] 的 LCS 的末尾。
    • 若 X[i−1]≠Y[j−1]X[i-1] \neq Y[j-1]X[i−1]=Y[j−1]，則 dp[i][j]=max(dp[i−1][j],dp[i][j−1])dp[i][j] = max(dp[i-1][j], dp[i][j-1])dp[i][j]=max(dp[i−1][j],dp[i][j−1])，且 label[i][j]="↑"label[i][j] = "↑"label[i][j]="↑" 或 label[i][j]="←"label[i][j] = "←"label[i][j]="←"。代表不相等，則各退一步，取最大值。
  • 在重購 LCS 時，從 dp 中找到最長的 LCS 長度，並依照 labellabellabel 的方向，找到轉移來源，即可重構出對應的 LCS。
  • 時間複雜度：O(n2)O(n^2)O(n2)，其中排序的時間複雜度為 O(nlogn)O(nlogn)O(nlogn)，LCS 的時間複雜度為 O(n2)O(n^2)O(n2)。
  • 空間複雜度：O(n2)O(n^2)O(n2)， dpdpdp 和 labellabellabel 各佔一個 n×mn \times mn×m 的二維陣列。

方法二：直接做 Dynamic Programming

相對直覺，可以從「求 LIS 長度」直接想到的方法。

  • 令 dp[i]dp[i]dp[i] 表示以 nums[i]nums[i]nums[i] 結尾的 LIS 長度；令 prev[i]prev[i]prev[i] 表示以 nums[i]nums[i]nums[i] 結尾的 LIS 的前一個位置。
  • 則 dp[i]=max(dp[i],dp[j]+1)dp[i] = max(dp[i], dp[j] + 1)dp[i]=max(dp[i],dp[j]+1) for j∈[0,i)j \in [0, i)j∈[0,i) if nums[j]<nums[i]nums[j] < nums[i]nums[j]<nums[i]。
    • 即枚舉所有位置 iii 以及 iii 前面的所有位置 jjj，若 nums[j]<nums[i]nums[j] < nums[i]nums[j]<nums[i]，則 nums[i]nums[i]nums[i] 可以接在 nums[j]nums[j]nums[j] 後面，並構成更長的 LIS ，更新 dp[i]dp[i]dp[i] 和 prev[i]prev[i]prev[i] 。
  • 在重購 LIS 時，從 dpdpdp 中找到最長的 LIS 長度，並從 prevprevprev 中重構出對應的 LIS (由後往前)。
  • 時間複雜度：O(n2)O(n^2)O(n2)
  • 空間複雜度：O(n)O(n)O(n)， dpdpdp 和 prevprevprev 各佔一個長度為 nnn 的陣列。

方法三：Greedy + Binary Search

CLRS Exercise 15.4-6⋆ 要求的 O(nlogn) 時間複雜度的演算法，即 Robinson-Schensted-Knuth Algorithm 。

  • 如果已經得到的 LIS 結尾的數越小，那麼在後面接上其他數，會有更大的可能構成一個長度更長的 LIS。也就是說，在長度固定的情況下，結尾越小越好。根據這個貪心思路，可以記錄以下資訊：
    • tail[i] 表示 長度為 i+1 的 LIS 的最後一個元素的 最小值 ，初始化 tail[0] = nums[0]
      • 需要注意的是，tailtailtail 並不是LIS ，只是用來計算 LIS 長度的輔助陣列
    • pos[i] 紀錄 nums[i] 在 LIS 中的第幾個位置
  • 遍歷所有元素時，若 nums[i]>tail[−1]nums[i] > tail[-1]nums[i]>tail[−1] ，則可以構成更長的 LIS，將其接在 tailtailtail 的末尾，並更新 pos[i]pos[i]pos[i]。
  • 否則，在 tailtailtail 中二分查找，找到第一個大於等於 nums[i]nums[i]nums[i] 的元素，並將其更新為 nums[i]nums[i]nums[i]。
  • 重構時，從 pospospos 中找到最後一個元素，並從後往前重構 LIS。
  • 時間複雜度：O(nlogn)O(nlogn)O(nlogn)，遍歷 nnn 個元素，每次二分查找的時間複雜度為 O(logn)O(logn)O(logn)。
  • 空間複雜度：O(n)O(n)O(n)， tailtailtail 和 pospospos 各佔一個長度為 nnn 的陣列。
